# Video Upload

Use this skill to turn a local video file into a Cloudflare Stream watch link and add that link to the right GitHub surface.

## Workflow

1. Locate or produce the local video file to upload.
2. Run `scripts/upload_cloudflare_stream.py` with the Cloudflare API token, account id, and local video path.
3. Capture the printed watch URL.
4. Update the target PR or task with that URL.

Prefer updating PR bodies through `gh api .../pulls/<number> -X PATCH` instead of relying on `gh pr edit` for merged PRs.

## Script

Run:

```bash
python3 scripts/upload_cloudflare_stream.py \
  --api-token "$CF_API_TOKEN" \
  --account-id "$CF_ACCOUNT_ID" \
  --video-path /absolute/path/to/video.webm \
  --name "short-demo-name" \
  --json
```

Behavior:

- Creates a Cloudflare Stream direct upload session
- Uploads the local file
- Polls until `readyToStream=true`
- Prints the final watch URL, or JSON when `--json` is set

Key flags:

- `--api-token`: required
- `--account-id`: required
- `--video-path`: required local file path
- `--name`: optional display name in Stream
- `--max-duration-seconds`: optional, defaults to `3600`
- `--timeout-seconds`: optional readiness timeout, defaults to `180`

## PR Update Pattern

For open or merged GitHub PRs, prefer:

1. Read the current body with `gh pr view <number> --repo <owner/repo> --json body`
2. Append or replace a `## Demo Video` section
3. Patch the PR with:

```bash
gh api repos/<owner>/<repo>/pulls/<number> -X PATCH --input <json-file>
```

Where the JSON file looks like:

```json
{"body":"...updated markdown..."}
```

## Notes

- Assume the API token is already available in the current context or provided by the user; do not hardcode it into repo files.
- Prefer `.webm` or `.mp4` artifacts generated by Playwright or browser recording tools.
- If the user only wants the link, do not merge or modify code PRs beyond adding the hosted video URL they asked for.
